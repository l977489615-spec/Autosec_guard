# 智驭安盾 (SmartDrive Shield) 功能说明书

## 1. 项目简介
**智驭安盾 (SmartDrive Shield)** 是一款面向智能网联车辆 (ICV) 的自动化安全漏洞验证平台。系统集成真实物理探测能力、数字自动化与基于大语言模型 (LLM) 的分析能力。依靠模块化的 PoC 测试框架、Agentic AI（多智能体协作）与自动合规报告生成引擎，为汽车网络安全渗透测试提供了一站式、全要素的自动化解决方案。

## 2. 核心功能及架构

### 2.1 大规模物理 PoC 检测框架
- **海量安全检测矩阵**：默认配备了 71 项原生的真实验证脚本 (PoC)，采用 Python 原生库编写以开展真实的智能汽车网卡和总线数据包交互。
- **非破坏性防护机制**：系统安全引擎监控每一次探测，利用极简验证包（单次仅发送 1~3 个探测包）测试策略，最大程度确保台架及实际汽车控制系统不产生失能或崩溃。
- **按需精准测试**：提供按 IP 地址、蓝牙 MAC、无线网卡及 CAN 频道组合输入的灵活探测能力。对于当前未配置外设资产或测试目标缺失的组件，自动过滤并跳过无谓的诊断扫描。

### 2.2 多智能体协作渗透系统 (Agent Orchestrator)
本平台搭载了基于 MCP (Model Context Protocol) 协议构建的特化大模型 Agent 群体，通过动态任务流完成如同安全专家主导般的自主渗透作业：
1. **侦察 Agent (Reconnaissance)**：调动网络测绘模块扫测目标，梳理资产拓扑结构，自动定位开放网络服务端口及汽车内部可能的安全网关 (SEC-GW)。
2. **决策 Agent (Decision)**：大脑级调度枢纽。基于测绘数据、系统目标环境指纹、可用测试接口配置参数，从 70+ 个验证库中排查出有效和适用的验证漏洞链条，生成有序的执行路径计划。
3. **Weaponize Agent (未知服务动态探测系统)**：在决策层遇到现有静态扫描规则未触及的异常应用、未知新协议时激活介入。实时生成协议感知型动态探测 Python 脚本，先建立正常响应基线，再对异常响应、错误栈、调试泄露和可复现行为差异进行证据分级，突破常规扫描器的静态局限性。
4. **执行 Agent (Executor)**：接管测试计划并控制底层脚本执行框架。驱动 PoC 漏洞利用链与系统互动，截获详细返回载荷，根据临时测试状况进行动态策略回环调整（中断或重试），并详尽记录取证数据信息。
5. **评估 Agent (Assessment)**：负责归纳和出卷。依据前序收集和验证的真实取证数据，以 ISO/SAE 21434 和 UN R155/CSMS 等主流汽车工业合规条例的语言尺度进行总结评估，零幻觉出具整体风险评分。

### 2.3 自动合规审计报告引擎
- **标准级安全出卷**：由云端大模型 API（如 Qwen 千问大模型）实时构建严谨专业的中文安全评估报告，杜绝结果中掺杂“可能存在漏洞”的模糊臆造描述，确保每条评估内容 100% 紧扣实际执行阶段获取的证据载荷。
- **ISO 21434 TARA 矩阵匹配**：针对发现确认的真实漏洞缺陷，报告模块内会自动构建符合 TARA 的威胁分析及风险评估表格，科学拆解 CVSS 评分因子及实际安全业务影响定级。
- **一键高清归档导出**：系统底层打通了基于 Markdown 排版的 AI 结果，可以在前端控制台页面中一键生成清晰且带正反目录排版的高质量 PDF 审核报告，便于内部传递和长效合规审查入库。

### 2.4 可视化控制台与多端管控
- **全局态势感知大屏 (Dashboard)**：首屏直观展现被测车辆及整体系统的漏洞分布版图视图、风险严重性统计分布及引擎任务的并发执行运行状态。
- **双模弹性操作层**：系统支持双模切换运行，用户可一扫即用的形式启动全局链式自动化探测 (**Global Auto Scan**)，也可随时进入针对性的单一漏洞模块诊断详情页手工调整核心发包参数 (**Manual Diagnostic**)。
- **操作溯源与维基百科**：平台搭载完备的用户认证体系支持基于角色的权限控制 (RBAC) 进行分权分责；全局追踪溯源过往的每一次下发扫描任务的运行状态时间与配置。并内置完备的 71 个汽车漏洞维基百科页面，支持工程师全文索引翻阅与技术学习。

## 3. 六大安全维度的实体攻击面覆盖
智驭安盾从完整生命系统切入，囊括对车辆软硬件 6 大不同层级的细粒度攻防验证：
1. **网络侦察与暴露面情报**：扫描服务探活、T-Box 及网关边界暴露面数据收集、应用服务指纹爬取与蓝牙周边信标枚举。
2. **车内网络及应用基础服务**：SSH 硬编码弱口令猜解、测试调试接口暴露 (如 ADB)、FTP 文件系统未授权漫游、以及汽车主干网中 SOME/IP 等核心协议的异常鉴权和服务发现漏洞的验证。
3. **CAN 总线控制及底层诊断协议 (最核心)**：针对车载底层主流 CAN 总线，涵盖敏感消息嗅探与欺骗重放、UDS 协议异常交互（例如突破诊断会话鉴权、未授权控制下的控制器复位指令下发、内存总线的非法溢出获取及服务安全访问算法爆破）。
4. **无线通信射频与空口注入**：涵盖了针对各类外围通讯介质及底层组件的问题反馈，例如蓝牙模块的严重溢出 (包含 BLUFFS 密钥降级问题与 L2CAP DoS 崩溃利用)、Wi-Fi 无线连接点的 Evil Twin (邪恶双子) 连接劫持欺骗。
5. **车机交互应用层与生态软件设施**：涉及了智能座舱屏幕上的各类潜在风险，比如利用无线多媒体应用对底层展开 DoS 退化干涉 (如 AirPlay 内存溢出、CarPlay 流媒体解析崩溃)，以及通过车载 Webview 浏览器组件泄露内网敏感结构数据和座舱指令执行漏洞。
6. **高阶段组合型攻击与高阶固件验证**：拦截车载后台 OTA 后门中间人包投递、测试车辆电子钥匙的射频防盗信号被抓取和二次克隆回放的缺陷，以及测试控制台对于车检及固件未认证非法替换加载与 GPS 导航劫持注入反馈情况。
